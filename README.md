# 高性能 C++ RPC 框架

## 1. 项目概述

本项目是一个从零开始、使用现代 C++ 构建的轻量级、高性能远程过程调用（RPC）框架。它为构建分布式客户端-服务器应用程序提供了一个坚实的基础，重点强调了效率、并发性和可扩展性。

该框架允许客户端像调用本地函数一样执行远程服务器上的函数。它包括一个能够处理大量并发连接的多线程服务器、一个用于发送请求的客户端库，以及一套高效的二进制通信协议。

## 2. 技术亮点与核心特性

该框架在设计时充分考虑了性能和现代 C++ 的最佳实践。其核心技术特性如下：

* **异步网络 I/O**: 利用 **Boost.Asio** 库实现了一个非阻塞、事件驱动的网络模型（遵循 Proactor 设计模式）。这使得服务器可以用少量的 I/O 线程高效地处理大量并发连接。

* **并发请求处理**: 采用了一个**自定义的线程池** (`ThreadPool`) 来处理传入的 RPC 请求。这种设计将网络 I/O 操作与业务逻辑处理解耦，最大限度地提高了吞吐量，并确保服务器在高负载下也能保持响应。

* **高效的零拷贝缓冲区管理**: 实现了一个专为网络通信设计的自定义 `Buffer` 类。其关键创新在于设计了 `prependable` 预留空间，允许在消息体前添加头部，而无需复制或重新分配内存。这是一个显著的优化，最大限度地减少了内存操作。

* **自定义二进制协议与字节序安全**: 定义了一个简单高效的二进制协议，使用固定大小的头部 (`RpcHeader`) 将请求路由到正确的服务和方法。框架正确处理了主机字节序和网络字节序之间的转换，确保了跨平台的兼容性。

* **可插拔的数据序列化**: 集成了 **Google Protocol Buffers (Protobuf)** 来序列化应用层的请求和响应数据。Protobuf 是一种行业标准的、由 schema 驱动的格式，以其高性能和紧凑的数据体积而闻名。

* **现代 C++ 与可扩展设计**:
    * 采用**现代 C++** 编写，充分利用了智能指针 (`std::unique_ptr`, `std::shared_ptr`)、移动语义、`std::span` 和 lambda 表达式等特性，实现了安全、整洁且高效的代码。
    * 服务器围绕一个抽象的 `IService` 接口进行设计，使得在不修改框架核心代码的情况下，可以轻松地定义和注册新的服务。

## 3. 架构与请求生命周期

该框架由几个关键组件构成，它们协同工作以完成一次 RPC 调用。

#### 核心组件:
* **`RpcServer`**: 核心组件，负责管理已注册的服务，并使用 `ThreadPool` 来分发和处理请求。
* **`Session`**: 代表单个客户端连接的对象。它负责从套接字异步读取数据、解析完整的消息，并将响应写回客户端。
* **`ThreadPool`**: 管理一个工作线程池，负责执行具体的 RPC 业务逻辑，从而避免阻塞主 I/O 线程。
* **`Buffer`**: 自定义的内存缓冲区，在整个框架中用于高效地读取、写入和管理网络数据。
* **`RpcClient`**: 一个同步客户端，用于连接服务器、序列化请求和反序列化响应。

#### 请求生命周期:
1.  **客户端**: 请求使用 Protocol Buffers 进行序列化。序列化后的数据（payload）被写入一个 `Buffer`，然后包含路由信息和总长度的二进制 `RpcHeader` 被预置（prepend）到缓冲区的预留空间中。
2.  **服务器 (I/O)**: `Session` 对象异步地将数据流读入其内部的 `Buffer`。它首先等待接收完整的头部，然后根据头部中指定的长度读取剩余的消息体。
3.  **分发**: 一旦收到完整的消息，`Session` 将请求传递给 `RpcServer`，后者向 `ThreadPool` 中添加一个任务。
4.  **服务器 (工作线程)**: 线程池中的一个工作线程获取该任务。它反序列化请求，调用已注册的 `IService` 中的相应方法，并获得结果。
5.  **响应**: 结果被序列化回一个 `Buffer`，并预置上响应头部，然后将数据交还给 `Session`。
6.  **服务器 (I/O)**: `Session` 使用 `boost::asio::strand` 来确保线程安全地将响应缓冲区异步写回客户端套接字。

## 4. 关键技术栈

* **编程语言**: C/C++ 
* **网络库**: Boost.Asio
* **序列化**: Google Protocol Buffers
* **并发**: C++ 标准库 (`<thread>`, `<mutex>`, `<condition_variable>`, `<future>`)
